# bill-split

一个专门用于 `H女士` 和 `Daisy` 两人账单分摊的手机端轻量应用项目。它参考 `trip-splitter` 的实现方式，保留行程管理、OCR 识别、一句话计算和本地结算能力，目标是把外卖、餐饮、出游、团购、交通、住宿等账单快速整理成清晰的转账结果。

## 项目定位

首版只服务两个人：`H女士` 和 `Daisy`。不做多人扩展，不做复杂协作，不接支付宝或微信 API。核心流程是：

1. 先创建行程，例如 `6月8日周末游玩`
2. 进入行程后创建账单
3. OCR 识别账单截图，提取日期、场景、金额、消费物品、交易备注等信息
4. 输入一句话分摊说明，提取两人的分摊金额或消费权重
5. 手动校正确认字段并保存
6. 自动计算当前行程总金额、H女士合计、Daisy 合计和最终转账结果
7. 生成可复制或分享的行程账单分析

## 首版目标

- 创建行程，例如一次出游、一次聚餐、一个外卖账单集合
- 固定支持 H女士 和 Daisy 两人
- 逐笔记录账单：日期、场景、描述、总金额、付款人、H女士应收、Daisy 应收、交易备注
- 保留一句话计算功能
- 保留 OCR 识别账单截图功能
- OCR 提取结果可自动填入录入表单，并继续配合一句话计算两人分摊金额
- 支持默认均分、指定金额、按消费权重换算
- 支持凑单满减/叠券场景：按两人原价或凑单金额比例分摊券后实付
- 支持中文金额识别，例如 `十七块六毛八`、`壹佰贰拾叁元肆角伍分`
- 支持行程维度总金额、两人各自总金额和账单分析分享
- 本地保存账单数据，刷新后不丢失

## 推荐技术方案

首版使用纯 `HTML + CSS + JavaScript` 实现，无需后端和数据库，直接打开 `index.html` 即可运行。界面按手机端小程序/App 优先设计。

OCR 可继续使用 `Tesseract.js`。一句话计算先用规则解析实现，优先覆盖高频中文表达，不接大模型 API。

## MVP 验收标准

- 能录入 H女士 和 Daisy 的两人账单
- 能先创建行程，再进入行程创建账单
- 能从一句话中提取日期、总金额、两人应收金额、场景、交易备注
- 能理解凑单满减句式，并按实际付款金额换算两人分摊
- 能识别阿拉伯数字、中文口语金额和中文大写金额
- 能从账单截图中识别总金额、日期、场景、消费物品等信息
- OCR 结果能同步到录入表单，并继续使用一句话计算补全两人应收
- 能展示行程总金额、H女士合计、Daisy 合计
- 能复制或分享当前行程的账单分析
- 结果清晰展示“H女士需转给 Daisy”或“Daisy 需转给 H女士”
- 无需登录、无需支付平台 API、无需安装复杂依赖
